[ 장치 스프라이트 시트를 여기에 넣으세요 ]

★ 모든 장치는 "가로 4프레임" 시트로 움직입니다.
   파일 1개 = [프레임0][프레임1][프레임2][프레임3] (가로로 4칸 나열, 세로 1줄).
   장치의 실제 표시 크기(아래 괄호)를 한 프레임 크기로 잡고, 그걸 가로로 4번 이어붙이면 됩니다.
   예) 창고(144x144) → 시트 전체 576x144.

물류:
  belt.png       ← 컨베이어 벨트 (프레임 48x48, "동쪽→" 기준 / 게임이 방향 회전 + 4프레임 흐름)
  crossbelt.png  ← 횡단 벨트 (프레임 48x48)
  tunnel.png     ← 터널 1x5 (프레임 240x48, 동쪽 기준 / 입구→출구)
  sorter.png     ← 분류기 (장치 2칸. 프레임 48x96)
  grabber.png    ← 집게 (그래픽은 1x3 가로형! 한 프레임 = 가로로 긴 집게 한 마리, 권장 144x48)
                   → 시트 전체 = 144x48 프레임 4개 가로 나열 = 576x48
                   → 게임이 "원본 비율 유지"로 3칸 길이에 맞춰 그림(비율 달라도 안 찌그러짐)
  warehouse.png  ← 창고 (프레임 144x144)

생산:
  penbox.png     ← 우리 (★예외: 애니메이션 없음. 144x144을 48x48 9분할 = 모서리4·변4·중앙1로 울타리 타일링)
  birthing.png       ← 출산대(실장석 장착됨, 프레임 96x96)
  birthing_ready.png ← 출산대(비어있음 — 장착 전, 프레임 96x96)

가공:

  slaughter.png   ← 도축기   (프레임 144x144)
  deshell.png     ← 탈복기   (프레임 192x96)
  grinder.png     ← 분쇄기   (프레임 96x96)
  correction.png  ← 교정시설 (프레임 144x144 — 자실장 6마리 수용 우리)
  mixer.png       ← 배합기   (프레임 96x96)
  cookery.png     ← 조리실   (프레임 144x96)

특수(모두 1x1, 프레임 48x48. 고스트/설치 시 영향 범위가 노란 점선으로 표시됨):
  speaker.png     ← 태교 스피커 (3x3 범위 스탯↑)

* 한 프레임 크기는 자유 — 게임이 가로 4등분해서 사용. 파일 없으면 색 블록(벨트는 화살표 흐름)으로 표시.
* 프레임 속도: js/config.js 의 ANIM_FPS(기본 6). 파일명/크기는 G.DEVICES.
